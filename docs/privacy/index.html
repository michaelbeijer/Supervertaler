Privacy Policy

Last updated: 14 March 2026

The short version: Supervertaler does not collect, store, or transmit your data to Supervertaler servers. Your translations, termbases, and documents stay on your machine. When you choose to use AI features, data is sent directly from your computer to the AI provider you selected, using your own API key. We never see it.

This policy covers both Supervertaler Workbench (the desktop application) and Supervertaler for Trados (the Trados Studio plugin). We know that as a professional translator you often work with confidential client material, and we have designed our software with that in mind.

1. Data that stays on your machine

The following data is created and stored locally on your computer only. It is never transmitted anywhere:

  • Translation files — your SDLXLIFF, DOCX, and other documents remain entirely on your local file system
  • Termbases — your Supervertaler glossary databases (.db files) and MultiTerm termbases (.sdltb files)
  • Translation memories — all TM data stays local
  • Settings and preferences — stored in your local AppData folder
  • Chat history — AI Assistant conversations are held in memory only during your session and are never written to disk

2. Data sent to AI providers (only when you use AI features)

When you use the AI Assistant, Batch Translate, or single-segment AI translation, data is sent to the AI provider you have chosen and configured. This only happens when you explicitly trigger an AI action — never in the background.

What is sent

Depending on the feature and your settings, the following may be included in AI requests:

  • Source text segments (the content you are translating)
  • Target text segments (your existing translations, for context)
  • Terminology from your termbases (source/target term pairs)
  • Translation memory matches (if enabled in AI Context settings)
  • Surrounding segments for document context (configurable)
  • Project and file name, source/target language pair
  • Images you attach in the AI Assistant chat
  • Your chat messages and conversation history (current session only)

Where it is sent

Provider Destination Your data is governed by
OpenAI api.openai.com OpenAI API Data Usage Policy
Anthropic (Claude) api.anthropic.com Anthropic Privacy Policy
Google (Gemini) generativelanguage.googleapis.com Google Gemini API Terms
Ollama (local) localhost (your own machine) No data leaves your computer
Custom endpoint Server URL you configure That provider's terms

For maximum confidentiality: If you work with sensitive or NDA-protected content and want AI features without any data leaving your machine, use Ollama as your provider. All processing happens locally on your own hardware.

What is NOT sent to AI providers

  • Your full documents — only the segments relevant to your current request
  • Your API keys for other providers
  • Your license key or account information
  • Any data from other projects or files not currently open

3. API keys

Your AI provider API keys are stored locally in your settings file (%LocalAppData%\Supervertaler.Trados\settings.json). They are sent only to the respective AI provider when making API calls. Supervertaler never transmits your API keys to any other destination.

4. Update checks

Supervertaler for Trados checks for new versions via the GitHub API (api.github.com) on startup. This request contains only a generic user-agent string (Supervertaler-Trados-UpdateCheck/1.0) — no personal information, usage data, or system details are sent. You can skip suggested updates or dismiss them.

5. License validation

If you have a paid subscription, your license is validated through Lemon Squeezy (api.lemonsqueezy.com). The only data sent is:

  • Your license key
  • An anonymous machine identifier (a one-way hash — it cannot be reversed to identify your computer)

No translation content, file names, usage statistics, or personal information is included in license checks.

6. What we do NOT do

  • We do not collect telemetry or usage analytics
  • We do not track which features you use or how often
  • We do not use cookies (this is a static website with no tracking)
  • We do not sell or share any data with third parties
  • We do not have access to your translations, termbases, or documents
  • We do not run any Supervertaler-operated servers that process your data

7. Open source transparency

Both Supervertaler products have their source code publicly available on GitHub. You can verify every network call the software makes by inspecting the code yourself:

  • Supervertaler Workbench source code
  • Supervertaler for Trados source code

8. Your responsibilities

When you use AI features with cloud-based providers (OpenAI, Anthropic, Google), the text you send is processed by those providers under their own privacy policies. If you are bound by NDAs or confidentiality agreements with your clients, please:

  • Review the data handling policy of your chosen AI provider
  • Consider using Ollama for fully offline, local-only AI processing
  • Adjust AI Context settings to limit what data is included in requests (e.g., disable TM matches or restrict the number of context segments)
  • Consult with your client if in doubt about whether AI-assisted translation is permitted

9. Contact

If you have questions about this privacy policy or how Supervertaler handles your data, please contact us at support@supervertaler.com.

10. Changes to this policy

We will update this page if our data practices change. The date at the top of this page indicates when the policy was last revised.
